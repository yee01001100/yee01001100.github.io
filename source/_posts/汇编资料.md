## 汇编指令与寄存器

### 前导

什么是CPU？

CPU是计算机的核心组件，负责执行计算机程序的所有**指令**。它的性能直接决定了计算机的运算速度和处理能力。

**指令**：CPU执行的指令是**机器指令**，它是由0和1组成的二进制代码。CPU能够直接理解和执行的指令就是机器语言，这些指令在底层以二进制的形式存在，是计算机硬件能够直接识别的最基本形式。

虽然计算机本身理解机器语言，但是这种语言直接使用0和1来编写的方式还是太困难和低效。为了方便程序员的操作，**汇编语言**应运而生。汇编语言是一种使用助记符来表示机器指令的语言。每条汇编指令直接映射到一条机器指令，但是计算机并不认识汇编，它就需要通过汇编器将其转化为机器语言。汇编器是将汇编代码翻译成二进制机器指令，来供CPU执行的。

举个🌰

汇编语言中的 `MOV EAX ，1`，会被汇编器转化为机器语言的二进制指令 `B8 01 00 00 00`，这条机器指令直接控制CPU执行相应的操作。

```asm
10111000 00000001 00000000 00000000 00000000
汇编代码形式
MOV EAX, 1
```

汇编语言与硬件架构紧密相关，不同的计算机架构（如x86，ARM)的汇编语言有很大不同

计算机架构是指计算机系统的设计架构，包括其处理器的内部结构，指令集（ISA），寄存器，内存模型等。每组架构定义了一组可执行的指令，程序员编写的汇编语言正是与这些指令集架构密切相关的。

每种 ISA：

- 有自己的一套 **寄存器名字**
- 自己的一套 **指令名字和用法**
- 自己的 **寻址方式、指令长度** 等特征

不同架构汇编指令的区分：（感兴趣的可以自行搜查）

1.x86 / x86-64（Intel/AMD PC 上最常见）

2.ARM / AArch64（手机、树莓派、Apple M 系列）

3.MIPS（一些嵌入式、教材常用）

4.RISC-V（近几年很火的开源指令集）

### 寄存器

寄存器是CPU内部的高速存储单元。它用于存放临时的数据和计算的中间结果，具有非常快的访问速度。

寄存器的大小通常与CPU的位数有关：

在32位的x86架构中，寄存器的大小是32位（4字节）（内存地址也是）

在64位的x86-64架构中，寄存器的大小是64位（8字节）（内存地址也是）

`EFLAGS` 是 **x86 架构**里的“标志寄存器”

作用：**记录算术/逻辑结果的状态**（供后续条件跳转用）

eflags 这里展开说一下

进位标志 CF (Carry Flag):  如果无符号数运算的结果的最高位产生了一个进位或者借位，那么其值为 1，否则为 0 

```asm
MOV EAX, 0xFFFFFFFF			;32 位无符号数的最大值
ADD EAX, 1					;EAX=0 ，并且产生一个进位（CF = 1）
JC overflow					;如果出现进位（Carry）就跳转
```

奇偶标志 PF (Parity Flag): 用于反应运算结果中一个数的奇偶性。如果低 8 位中 `1` 的个数是 **偶数** 则PF 的值为1，否则为 0

**零标志位 ZF (Zero Flag)**: 用来反映运算结果是否为 0。如果运算结果为 0 则其值为1，否则为 0

符号标志 SF (Sign Flag): 用来反映运算结果的符号位，它与运算结果的最高位相同。0 表示正数，1表示负数。

溢出标志 OF (Overflow Flag): 用于反映**有符号数加减运算所得结果是否溢出** 

运算结果超过当前运算位数所能表示的范围，则称为溢出。在有符号数的运算中，有如下规律：

溢出是运算的数学结果超出了当前数据类型能表示的数值范围

比如 8 位有符号数范围：`-128 ~ 127`

- 127 + 1 = 128（数学结果），但 128 超出范围 → 有符号溢出
- -128 - 1 = -129，也超出范围 → 有符号溢出

正+正 如果结果是负数，则说明有溢出

负+负 如果结果是正数，则说明有溢出

正+负 永远都不会有溢出

### 常见的汇编指令

#### nop

表示 No Operation，机器码为 0x90，没有操作数，执行这条指令时 CPU 什么事都不干。

#### push

PUSH 指令将数据压入栈中。栈指针寄存器 ESP(RSP) 自动递减。

#### pop

POP 指令从栈中弹数据。栈指针寄存器 ESP(RSP) 自动递增。

#### mov

MOV 指令将数据从源操作数移动到目的操作数。

指令格式为 mov x1, x2。意为从 x2 移动数据到  x1。

x1, x2 可以是：

- 寄存器到寄存器

- 内存到寄存器，寄存器到内存

- 立即数到寄存器，立即数到内存

不能从内存直接到内存。

例如：

```c
mov eax, 5	 //立即数到寄存器
mov eax, [0x12345678]	//内存到寄存器，汇编中用[地址A]来表示地址A存储的内容
mov eax, dword ptr [0x123456]	//从这个内存地址取出4字节到eax里
mov [0xdeadbeef], eax	//寄存器到内存
```

#### lea

LEA 指令用于取地址。

指令格式为 lea x1, x2。

x1 是 寄存器，x2 是内存地址，常用 [] 语法形式，意为引用求值。

常用于指针运算，有时用于数值计算。

例如

```c
内存[0x1010] = 0xDEADBEEF
rbx = 0x1010
rcx = 2
mov rax, [rbx] >去地址 0x1010 访问内存，把里面的值 0xDEADBEEF 读出来。即：rax = 0xDEADBEEF
lea rax, [rbx] >只取rbx的地址给rax。
lea rax, [rbx + rcx*4 + 8] -> rax = 0x1010 + 2*4 + 8 = 0x1020
```

### 算数运算指令

| 操作符              | 功能       | 示例         |
| ------------------- | ---------- | ------------ |
| add                 | 加法       | add eax,ebx  |
| sub                 | 减法       | sub eax,ecx  |
| mul                 | 无符号乘法 | mul ecx      |
| imul                | 有符号乘法 | imul eax,ecx |
| div                 | 无符号除法 | div ecx      |
| idiv                | 有符号除法 | idiv ecx     |
| inc（increase + 1） | 自增       | inc eax      |
| dec（decrease -1）  | 自减       | dec eax      |

#### add

ADD 指令计算加法操作。

指令格式为 add x1, x2。其作用是x1+=x2

x1，x2 可以是 寄存器，x2 还可以是立即数，但二者不能同时是内存操作数。

指令的计算结果将影响 eflags 寄存器，修改 OF, SF, ZF, AF, PF, CF 标志位。

```c
add esp, 8
add eax, ebx
add [eax], ebx
```

#### sub

SUB 指令计算减法操作。

指令格式为 sub x1, x2。其作用是x1-=x2

x1，x2 可以是寄存器，x2 还可以是立即数，但二者不能同时是内存操作数。

指令的计算结果将影响 eflags 寄存器，修改 OF, SF, ZF, AF, PF, CF 标志位。

```c
sub esp, 0xC
sub eax, ebx
sub [eax], ebx
```

#### IMUL/MUL

`IMUL`：**有符号**乘法（按照有符号数的语义判断溢出），有单操作数 / 双操作数 / 三操作数等多种格式。

三种指令格式：

单操作数：默认被乘数在EAX，得到的 64 位结果放到 **EDX:EAX** 里（高 32 位在 EDX，低 32 位在 EAX）

双操作数：imul dest ， src ，目标寄存器存储结果

三操作数： imul dest，src ，imm ，将src *imm 的结果存入 dest

![](https://cdn.jsdelivr.net/gh/seeker-fang/Images//202512031649416.jpeg)

MUL 指令是无符号乘法。

操作数：默认为EAX寄存器是被乘数，操作数是乘数

结果：如果是32位，低32位结果存储在EAX，高32位存储在EDX

```asm
mov eax, 5			;被乘数
mov ecx, 3			;乘数
mul ecx				;无符号乘法
;结果存储在EAX（低32位），高位结果存储在EDX
```

最后eax=15，edx=0（因为乘积小于2^32，edx为0）

**2 ^ 32 - 1 = FFFF FFFF**

#### IDIV/DIV

DIV指令用于实现无符号数的除法运算。(把操作数当作非负整数处理)

操作数：

- 被除数默认是EDX：EAX组成的64位整数（高32位在EDX，低32位在EAX）。

- 操作数为除数，结果：
  - 商存储在EAX
  - 余数存储在EDX

注意：需要确保EDX被清零（或正确设置），否则结果可能出错。

```asm
mov edx,0		;高位被除数清零
mov eax,15		;低位被除数
mov ecx,3		;除数
div ecx			;无符号除法，EAX = EAX / ECX，EDX = EAX % ECX
```

IDIV指令是实现有符号整数除法

操作数：

- 被除数为EDX：EAX组成的64位整数（同div）
- 操作数为除数，结果：
  - 商存储在EAX
  - 余数存储在EDX

这与div类似，也是需要正确设置EDX，且结果符号由操作数符号决定。确保EDX符号开启：cdq指令

```asm
mov eax, -15		;低位被除数
cdq					;扩展符号位到EDX（符号扩展指令）
mov ecx,3			;除数
idiv ecx			;有符号除法，EAX = EAX / ECX , EDX = EAX % ECX
```

结果：EAX = -5 （商）EDX=0（余数）

#### AND(与运算)

AND 指令实现操作数的逻辑与运算。指令格式为 and x1, x2。

x1, x2 可以是 r/m16, r/m32, r/m64, x2 还可以是立即数，但二者不能同时是内存操作数。

两个输入都为1，结果才为1，否则是0

#### OR(或运算)

OR 指令实现操作数的逻辑或运算。指令格式为 or x1, x2。

x1, x2 可以是 r/m16, r/m32, r/m64, x2 还可以是立即数，但二者不能同时是内存操作数。

两个输入都为0，结果才为0，否则是1

#### XOR

XOR 指令实现操作数的异或运算。指令格式为 xor x1, x2。

异或是同为0，异为1。 

x1, x2 可以是 r/m16, r/m32, r/m64, x2 还可以是立即数，但二者不能同时是内存操作数。注：XOR 常用于清零运算，如 xor eax, eax，效率高。

应用：

**1.加密解密**

明文 ^ key = 密文

密文 ^ key = 明文

**2.反转标志位**

```asm
xor eax, 0x1		;如果eax的最低位是1，变为0，如果是0，变为1
```

**3.清零寄存器**

```asm
xor eax, eax		;eax清零
```

#### NOT(非运算)

NOT 指令实现操作数的逻辑非运算。指令格式为 not x。 x 可以取 r/m16, r/m32, r/m64。

取反，0变1，1变0

#### SHL

SHL 指令用于实现操作数的逻辑左移运算。

指令格式为 shl x1, x2： x1 可以取 r/m16, r/m32, r/m64

x2 可以是 cl 或者占用 1 字节的立即数，表示左移长度

0001 1010 << 2 = 0110 1000

**x << k == x * 2^k**

#### SHR

SHR 指令用于实现操作数的逻辑右移运算。

指令格式为 shr x1, x2： 

x1 可以取 r/m16, r/m32, r/m64

x2 可以是 cl 或者占用 1 字节的立即数，表示右移长度

注：**右移运算是除以 2 的高效实现方法**，被用于除法性能优化。

### 逻辑比较指令

#### CMP

CMP 指令用于两个操作数的大小比较。

指令格式为 cmp x1, x2。

cmp 指令通过将 x1 减去 x2 的方式实现比较，根据减法结果设置 eflags 的相应标志位，且将减法结果丢弃。

cmp <-> sub ：

- 以相同的方式进行减法操作，以同样的方式设置 eflags 寄存器

- 但 sub 指令将计算结果存入 x1，而 cmp 指令不保存计算结果

修改的 eflags 寄存器标志位有 CF, OF, SF, ZF, AF, PF。     

```c
mov eax, 1
mov bl, 0x80
mov cl, 0x9
cmp bl, cl
jb label1	//小于跳转
mov eax, 0
label1：
...
```



#### TEST

TEST 指令用于两个操作数的逻辑比较。

指令格式为 test x1, x2。

test 指令通过计算两个操作数的逻辑与**(&)**位运算实现比较操作，根据与操作的结果设置 eflags 的相应标志位，且将与操作计算结果丢弃。

修改的 eflags 寄存器标志位有 SF, ZF, PF。

```c
mov eax, 1
mov ecx, 0x9
test ecx, ecx	//还是原来的0x9
jz label1
mov eax, 0
label1：
...
```

用途：常用于检查某位是否为0（如 TEST EAX，EAX 来判断EAX是否为0）。如果eax是0，0 and 0还是0，ZF=1

### 控制转移指令

#### JMP(无条件跳转)

JMP 指令改变 EIP(RIP) 的值到目标地址，实现控制流跳转。

指令格式为 jmp x。

x 可取 imm, r/m16, r/m32, r/m64 等。

jmp 目标地址 x 的主要形式：

- 短跳方式，使用相对地址

  - 相对于 jmp 指令的下一条指令，1 字节（-128 ~ 127）范围

  - 例如，地址0x00401021 处指令要表示 “jmp 0x00401028”为短跳方式，就是机器码 EB 05 。内存中并没有 0x00401023这一字符串或常量，而是前跳 5 个字节。  EB->jmp

    假设这里使用短跳（`EB xx`），指令起始地址：`instr = 0x00401021`。

    短跳的长度 = 2，所以**下一条指令地址**（next） = `instr + 2 = 0x00401023`。

    计算相对偏移：`rel = dest - next = 0x00401028 - 0x00401023 = 0x05`（十进制 +5）。

    因为 +5 在 -128..+127 范围内，可以用短跳。机器码就是 `EB 05`，存入内存的是这两个字节。

    CPU 执行时把 `next` 加上 `0x05` 得到 `0x00401028`，跳过去。

  - 某些反汇编器将其标记为 jmp short

- 近跳方式，使用相对地址

  - 相对于 jmp 指令的下一条指令，相对地址 4 字节范围

- 远跳方式（段间转移）

- 绝对地址，使用直接地址

  - 指令中硬编码方式的内存地址

- 绝对地址，使用间接地址

  - 指令中寄存器或内存间接访问的地址	

例如

```c
jmp eax
jmp 6	//短跳
jmp -2
jmp 0xdeadbeef
jmp [esp]	//内存间接访问
```



#### JCC(条件跳转)

Jcc 指令是一系列条件跳转指令的集合，该指令在指定条件满足时将控制流转向目标地址。

指令格式为 jcc x。

启动设置：

- 比较操作设置 eflags 寄存器的标志位
- 如 JNZ 指令，当 eflags 的 ZF 未被设置（ZF=0）时，指令满足跳转条件，执行控制转移到目标地

址 x 

几个常见 jcc 指令的判断条件：

- jz/je：if ZF == 1 （相等）跳转
- jnz/jne: if ZF == 0 ，（不等）跳转
- jg ：如果大于就跳转
- jl：如果小于就跳转
- jle/jng: if ZF == 1 or SF != OF
- jge/jnl: if SF == OF
- jbe: if CF == 1 or ZF == 1
- jb: if CF == 1

这些条件通过查看 eflags 寄存器的相应位进行判断。常用的设置 eflags 标志位的指令有 cmp, test运算指令等。记住几个单词的前缀就行
z：zero ，零

e：equal，相等

g：great，大

l：less，小

n：not，不

```c
mov bl, 0x60
mov eax, 1
sub bl, 0x31
jnz label1
mov eax, 0
label1:
...
```



#### CALL

CALL 指令将程序控制流转向一个子函数。

指令格式为 call x。

x 可取 r/m16, r/m32, r/m64, ptr16:16, ptr16:32。

启动设置：参数传递，使用函数调用约定。

具体过程：

1. 首先，将下一条指令的地址压入栈中，也就是push 返回地址
2. 然后，改变 EIP(RIP) 的值为 x，进入到call的内部执行，相当于jmp x
3. 执行完根据push的返回地址，返回到call下一行的执行代码（ret起作用）

x 可以是绝对地址，也可以是相对于 CALL 指令尾的相对地址

#### LEAVE

LEAVE 指令用于子函数退出时清理栈的操作，常与 RET 指令配套使用。

实际操作为：

```asm
MOV ESP EBP

POP EBP
```

1. 设置 ESP(RSP) 为 EBP(RBP) 

   将栈底指针的值赋值给栈顶指针，释放局部变量所占的栈空间。

2. 弹出栈顶数据到 EBP(RBP) 

   从栈中弹出保存的调用者的EBP值，恢复调用者的栈底指针。

#### RET

RET 指令将程序控制流从子函数返回到调用函数。

两种指令格式：

- ret
  - 将栈顶数据弹到 EIP(RIP)
  - 一般用于 cdecl
- ret x
  - 将栈顶数据弹到 EIP(RIP)
  - ESP(RSP) += x
  - 一般用于 stdcall

### 字符串操作指令

####  movs

movs 是一组字符串操作指令的通称，表示“移动字符串数据”。它可以根据操作的数据宽度分为以下几种具体形式：

1.movsb：移动一个字节（b表示byte，1字节）

2.movsw：移动一个字（w 表示word，2字节）

3.movsd：移动一个双字（d表示double word，4字节）

4.mosvq：（64位模式下可用）：移动一个四字（q表示quad word，8字节）

#### rep

**1.rep （repeat）**

- 作用重复执行后续字符串操作指令（如 movsb，stosb，lodsb等），直到ECX（或CX）寄存器的值减为0。或者说ECX不断减少，且>0时重复。
- 用法：不关心条件，仅重复执行

**2.`repe `或 `repz` (repeat while equal/repeat while zero)**

- 作用：当ZF=1且ECX不为0时，重复执行字符串操作指令。
- 常用于字符串比较指令（如cmpsb 或 scasb），表示“当相等时继续比较”。

 **3.repne 或 repnz（repeat while not equal / repeat while not zero）**

- 作用：当ZF为0且ECX不为0时，重复执行字符串操作指令。
- 常用于字符串操作指令，表示“当不相等时继续比较”。
- repne 和 repnz 是等价的。



**rep movsb 指令**

作用：批量复制字符串数据。

- rep 是重复指令前缀，结合movsb 一次复制一个字节。
- ECX 指定要复制的字节数，ESI和EDI分别为源地址和目标地址。

```asm
mov ecx, length			;设置要复制的字节数
mov esi, source			;源字符串地址
mov edi, destination	;目标字符串地址
rep movsb				;批量复制字符串
```



**scas 指令**

作用：扫描字符串中的特定字符。

- 扫描方向由DF（Direction Flag）决定，cld 清除DF从低地址扫描，std 设置DF 从高地址扫描。
- 用于实现字符查找（如 strchr）

```asm
mov edi, offset str_buffer		;EDI指向要扫描的字符串
mov al, 'A'						;查找字符‘A’
cld								;设置扫描方向为递增
mov ecx, length_of_string		;ECX设置为字符串的长度

repne scasb						;扫描直到找到‘A’或结束
jnz	not_found					;如果未找到A，跳转到 not_found
;找到‘A’后，继续处理
```

**stos 指令**

作用：向目标地址填充数据

- 通常用于清空字符串或填充特定值。

```asm
mov edi, offset_buffer		;目标地址
mov al, 0					;填充值为0
mov ecx, size				;填充大小
cld							;设置方向标志为递增
rep stosb					;填充字符串
```

用途：实现 memset 或者清零操作。

**loads 指令**

lodsb：加载一个字节到 AL，并将 ESI 自增1。

lodsw：加载一个字到 AX，并将 ESI 自增2。

lodsd：加载一个双字到 EAX，并将 ESI 自增4。

其中lodsb常与stosb一起使用
